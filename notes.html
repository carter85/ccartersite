Words to understand for chapter 1


Script

A Script is a series of instructions that a computer can follow to achieve a goal. It can be compared to recipes, handbooks, or manuals.

Vocabulary

The words that computers can understand.

Syntax

How words are put together to create instructions a computer can follow.

Object

In computer programming, each physical thing in the world can be represented as an object.

Properties

Properties describe the characteristics of an object. Each property has a name and a value, and each tell you something about the object.

Interpreter

The interpreter takes instructions and translates them into instructions the computer can use to achieve the tasks you want it to perform.

Interpreted Programming Language

An interpreted programming language, like JS, in which each line of code is translated one-by-one as the script is run.

Events

An event is something that hapens on a computer that the computer can react to, such as a keypress or moving the mouse over an object.

Methods

Methods represent things that people need to do with objects. They can retrieve or update the values of an objects properties. You do not always need to know how a method achieves a task, but you need to know how to ask the question and how to interpret any ansers ig gives you.

Calling

Programmers refer to using a method of an object as calling.

.

The full stop is referred to as the member operator. The document object has has several methods and properties, and are known as members of that object. You access the members of an object using a dot between the object name and the member you want to access.

Seperation of Concerns

When possible, aim to keep the three languages in seperate files, with the HTML page linking to the CSS and JS files. This is referred to as the seperation of concerns.

Content Layer

Where the content of the page lives, the HTML gives the page structure and adds semantics. Stored in .html files

Presentation layer

The CSS enhances the HTML page with rules that state how the HTML content is presented. Stored in .css files.

Behavior Layer

Where the behavior of the page can be changed, adding interactivity. Stored in .js files.

Progressive Enhancement

Progressive enhancement is the strategy of implementing HTML first, focusing on content, followed by CSS to improve the look of the page, and then JavaScript, adding interactivity.

Document

The document refers to the entire web page.


Properties

Properties describe characteristics of the current web page, such as the title.

Methods

Methods perform tasks associated with the document currently loaded in the browser, such as getting information from a specified element or adding new content.

Events

You can respond to events, such as a user clicking or taping on an element.

Document Object Model(DOM)

When a browser creates a model of a web page, it also creates a new object for each element on the page, together these objects are described as the Document Object Model(DOM).

Writing a Script

To write a script, you need to first state your goal and then list the tasks that need to be completed in order to achieve it.

A computer doesn't perform tasks like a person; it needs to follow instructions every time it performs a task, so a program must give the computer enough detail to perform the task as if every time were its first time.

Start with the big picture of what you want to achieve, and break that down into smaller steps:
  • Define the goal
  • Design the script
  • Code each step

Designing a Script: Tasks

>Once you know the goal of your script, you can work out the individual tasks needed to achieve it.

Designing a Script: Steps

Each individual task my be broken down into a sequence of steps. When you are ready to code the script, these steps can then be translated into individual lines of code.

From Steps to Code

Every step for every task shown in a flowchart needs to be written in a language the computer can understand and follow.

Computers solve problems programmatically: they follow a series of instructions one after another.

Defining a goal and defining the script

Consider how you might aproach a different type of script. Details goals for the script. Break it into a series of tasks that have to be performed in order to achieve the goals.

Often scripts will need to perform different tasks in different situations. You can use flowcharts to work out how the tasks fit together. The flowcharts show the paths between each step. Arrows show how the script moves from one task to the next. Different shapes represent different types of tasks. In some places are decisions which cause the code to follow different paths.

SUMMARY

  • A script is a series of instructions that the computer can follow in order to schieve a goal.
  • Each time the script runs, it might only use a subset of all the instructions.
  • Computers aproach tasks in a different way than humans, so your instructions must let the computer solve the task programmatically.
  • To aproach writing a script, break down your goal into a series of tasks and then work out each step needed to complete that task(a flowchart can help).

How A Browser Sees A Webpage

In order to understand how you can change the content of an HTML page using JS, you need to know how a broswer interprets the HTML code and aplies styling to it.

1. Receive a page as HTML code.

2. Create a model of the page and store it in memory. The model of a webpage can look like a family tree, at the top of which is a document object which represents the whole document. Beneath the document object, each object is stored in what is called a node. Each node represents another object, and these nodes are worked with to change objects in the DOM.

3. Use a rendering engine to show the page on screen. If these is no CSS, the rendering engine will aply default styles to HTML elements.

All major broswers use a JS interpreter to translate your instructions in JS into instructions the computer can follow.

SUMMARY

Computers create models of the world using data.

The models use objects to represent physical things. Objects can have: properties that tell us about the object; methods that perform tasks using the properties of that object; events which are triggered when a user interacts with the computer.

programmers can write code to say "When this event occurs, run that code."

Web browsers use HTML markup to create a model of the web page. Each element creates its own node (which is a kind of object).

To make web pages interactive, you write code that uses the browser's model of the web page.

The script element can be moved around on the page, and this affects where the new greeting is written into the page.

SUMMARY

It is best to keep JS code in its own JS file. JS files are text files that have the .js extension.

The HTML script element is used in HTML pages to tell the browser to load the JS file.
